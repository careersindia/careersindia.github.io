Number Analogy

Core Concept: Priority of Logical Patterns

Question

When solving number analogies, in what order should we check for different mathematical relationships?

Answer

The speaker recommends a specific order of checking for patterns to find the solution efficiently.

Explanation

The recommended priority for checking patterns is:

  1. Prime Numbers: Check if the numbers are prime.
  2. Cubes and Cube Roots (n³): Look for relationships involving cubes (e.g., n³ or n³ ± 1).
  3. Squares and Square Roots (n²): Look for relationships involving squares (e.g., n² or n² ± 1).
  4. Multiplication and Division (× / ÷): Check for simple multiplication or division relationships.
  5. Addition and Subtraction (+ / -): Check for simple addition or subtraction relationships.
  6. Digit Logic: If none of the above work, look for operations on the individual digits of the number (e.g., sum of digits, product of digits).

Important Notes: The speaker emphasizes, "This priority order is the key to solving analogies quickly. Most students start with addition/subtraction, but squares and cubes are far more common in exams. Always check for cubes and squares first! This will save you a lot of time and help you find the intended logic faster."

Question 1: Square Relationship (n² : n)

Question

49 : 7 :: 64 : ?

Answer

8

Explanation

The relationship between the first pair is that the second number is the square root of the first number (√49 = 7). Alternatively, the first number is the square of the second (7² = 49). Applying the same logic to the second pair, the square root of 64 is 8.

Question 2: Cube Relationship (n : n³)

Question

3 : 27 :: 4 : ?

Answer

64

Explanation

The relationship is that the second number is the cube of the first number. 3³ = 27. Therefore, we must find the cube of the third number, which is 4³ = 64.

Question 3: n² + 1 Relationship

Question

7 : 50 :: 11 : ?

Answer

122

Explanation

The pattern is n : n² + 1. For the first pair, 7² + 1 = 49 + 1 = 50. Applying this to the second pair, we get 11² + 1 = 121 + 1 = 122.

Important Notes: The speaker points out, "This is a very common pattern. You see the number 50 and you should immediately think of squares, specifically 7². The '+1' is a small extra step. Always be on the lookout for numbers that are very close to perfect squares or cubes."

Question 4: n³ - 1 Relationship

Question

5 : 124 :: 7 : ?

Answer

342

Explanation

The pattern is n : n³ - 1. For the first pair, 5³ - 1 = 125 - 1 = 124. Applying this to the second pair, we get 7³ - 1 = 343 - 1 = 342.

Question 5: n : n² - n Relationship

Question

8 : 56 :: 9 : ?

Answer

72

Explanation

There are two possible logics here, both leading to the same answer:

  1. Logic 1 (Multiplication): The second number is the first number multiplied by the number before it. 8 * 7 = 56. Therefore, 9 * 8 = 72.
  2. Logic 2 (Square - n): The pattern is n : n² - n. For the first pair, 8² - 8 = 64 - 8 = 56. For the second pair, 9² - 9 = 81 - 9 = 72.

Important Notes: "Sometimes there can be two or more correct logics," the speaker explains. "As long as they both lead to the same answer in the options, either one is fine. `n * (n-1)` is the same as `n² - n`. The multiplication logic is often faster to spot."

Question 6: Digit Sum Logic

Question

63 : 9 :: 68 : ?

Answer

14

Explanation

After checking for squares, cubes, and simple arithmetic, which don't fit, we move to digit logic. The relationship is that the second number is the sum of the digits of the first number. For the first pair, 6 + 3 = 9. Applying this to the second number, the sum of the digits of 68 is 6 + 8 = 14.

Important Notes: "Digit logic is your last resort," the speaker advises. "Only check for sum or product of digits after you have exhausted all the higher priority patterns like squares, cubes, and multiplication. In exams, the simpler arithmetic or square/cube patterns are more common."
